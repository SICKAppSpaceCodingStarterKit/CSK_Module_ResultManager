Here you can configure parameters to receive from other modules by their events to process them.
As events can have multiple parameters, it is possible to define the position of the parameter to use.
If you activate 'Merge data', it will only collect the data of all configured events and forwards them as a new event. No expression will be used.
INFO: In general, it will always wait to receive all configured events before it will process them. So make sure that the timing of the events is valid. Expression to process, like:
'param1 * (10+param2)' or 'param1 && param2' etc.
see also www.toggledbits.com/luaxp

Furthermore there are additional syntaxes available:
INFO: Please be aware, that following functions can not be combined with others (beside of 'dateTime' and 'timestamp'). If this is needed, just create an additional expression and link them... 'stringPack' : returns a binary string (see further info )

Sample:
stringPack('I2', param1) or
stringPack('I2I4I2', param1, param2, param3)
INFO: If you make use of multiple parameters, it is necessary to have them in a full and increasing order, like shown in the sample. 'stringUnpack': returns the value packed in binary string (see further info )

Sample:
stringUnpack('I2', param1) or
stringUnpack('I4', param1, 1,4)

The additional parameters are only needed if it is necessary to substract a substring out of the received value, e.g. here it will cut the binary string from position 1 to 4.
INFO: 'stringUnpack' only works with one single parameter! 'searchAndCut': searches for 2 strings in the received data as start and end positions and cuts the part between these positions. It is possible to add 2 additional numeric values to extend/reduce the positions.

Sample:
searchAndCut(param1, 'Value1', '}') or
searchAndCut(param1, 'Value1', '}', 1, -2)
'dateTime': returns Date and Time in string format = 'YYYY-MM-DDTHH_MM_SS', like '20240709T14_12_59'

'timestamp': returns a timestamp as string, like '2265809'
